# 프로젝트 리뷰어

## 개요

신속한 사용자 온보딩을 위한 포괄적인 프로젝트 리뷰 문서를 생성합니다. 효율적인 코드베이스 분석을 위해 serena MCP 도구를 사용하고 `/review` 디렉토리에 4개의 간결한 문서를 생성합니다: 프로젝트 개요, 구조 개요, 흐름 다이어그램, 사전 테스트된 빠른 시작 가이드.

## 이 스킬을 사용하는 시점

다음과 같은 명시적인 프로젝트 리뷰 요청 시 이 스킬을 사용합니다:
- "프로젝트 리뷰해줘"
- "프로젝트 리뷰 작성해줘"
- "프로젝트 검토"
- "프로젝트 리뷰 문서 생성"

## 리뷰 워크플로우

### 1. 초기 분석

시작하기 전에 효율적인 탐색을 위해 serena MCP 도구를 사용합니다:

```
mcp__serena-mcp__list_memories
mcp__serena-mcp__list_dir (relative_path: ".", recursive: true)
mcp__serena-mcp__get_symbols_overview
mcp__serena-mcp__find_file
```

분석 항목:
- 프로젝트 루트 디렉토리 구조
- 주요 설정 파일 (package.json, requirements.txt 등)
- 메인 진입점 및 핵심 모듈
- 기존 문서 및 README 파일
- 프로젝트 컨텍스트를 위한 사용 가능한 메모리

### 2. 리뷰 문서 생성

`references/review_templates.md`의 템플릿을 따라 `/review` 디렉토리에 4개의 문서를 생성합니다:

#### a. project_brief.md
- **목적**: 초간결 프로젝트 개요
- **형식**: 불릿 포인트만 사용
- **내용**:
  - 프로젝트 이름 및 주요 목적
  - 핵심 기술 스택
  - 주요 기능 (최대 3-5개 항목)
  - 대상 사용 사례
- **길이**: < 200단어

#### b. project_structure.md
- **목적**: 디렉토리 구조 개요
- **형식**: 간단한 주석이 포함된 트리 구조
- **내용**:
  - 주요 디렉토리 및 그 목적
  - 중요한 파일 및 그 역할
  - 모듈 구성
- **스타일**: 인라인 주석과 함께 트리 형식 사용
- **길이**: 상위 2-3 레벨에 집중, 중요한 경로만 주석 추가

#### c. project_flow.md
- **목적**: 시스템 아키텍처 및 상호작용 시각화
- **형식**: Mermaid 다이어그램
- **내용**:
  - 데이터 흐름 다이어그램
  - 모듈 상호작용 다이어그램
  - 요청/응답 흐름 (해당하는 경우)
  - 주요 프로세스 워크플로우
- **요구사항**: 모든 다이어그램은 mermaid 문법 사용
- **품질**: 실제 관계를 검증하기 위해 `mcp__serena-mcp__find_referencing_symbols` 사용

#### d. quick_start.md
- **목적**: 사전 테스트된 설정 및 실행 지침
- **형식**: 명령어가 포함된 번호 매기기 단계
- **내용**:
  - 환경 설정 (사전 요구사항, 가상 환경, 의존성)
  - 구성 단계
  - 개발 서버/빌드 명령어
  - 일반적인 개발 작업
- **중요**: 문서화하기 전에 모든 명령어 테스트 필수
  - 테스트 환경에서 설정 명령어 실행
  - 빌드 및 실행 프로세스 작동 확인
  - 실제 출력/동작 문서화
  - 환경별 요구사항 기록

### 3. 테스트 요구사항

quick_start.md 최종화하기 전:

1. **환경 설정 테스트**:
   ```bash
   # 가상 환경 생성 테스트
   # 의존성 설치 테스트
   # 구성 파일 확인
   ```

2. **빌드 및 실행 테스트**:
   ```bash
   # 개발 서버 시작 테스트
   # 빌드 프로세스 테스트
   # 일반 스크립트 작동 확인
   ```

3. **문서화**:
   - 실제 명령어 출력 스니펫 포함
   - 예상되는 동작 기록
   - 경고 또는 일반적인 문제 문서화
   - 테스트된 환경 명시 (Python 버전, OS 등)

### 4. 품질 기준

모든 문서는 다음 규칙을 따라야 합니다:

**형식**:
- 불릿 포인트 및 개요 형식만 사용
- 긴 단락 금지
- 불릿에 `-` 사용, 소제목에 `###` 사용
- 가능한 경우 각 불릿을 한 줄로 유지

**간결성**:
- 프로젝트 개요: < 200단어
- 프로젝트 구조: 상위 2-3 디렉토리 레벨에 집중
- 프로젝트 흐름: 최대 2-4개의 mermaid 다이어그램
- 빠른 시작: 필수 단계만 (< 15단계)

**정확성**:
- 정보 검증을 위해 serena MCP 도구 사용
- 문서화하기 전에 모든 명령어 테스트
- 실제 파일 경로 및 심볼 참조
- `find_referencing_symbols`로 모듈 관계 검증

## 출력 구조

정확히 다음 디렉토리 구조를 생성합니다:

```
/review/
├── project_brief.md        # 초간결 개요
├── project_structure.md    # 주석이 포함된 디렉토리 트리
├── project_flow.md         # Mermaid 다이어그램
└── quick_start.md          # 사전 테스트된 설정 가이드
```

## 도구 사용 전략

serena MCP 도구로 효율성을 극대화합니다:

1. **구조 분석용**:
   - 디렉토리 트리를 위한 `recursive: true`로 `list_dir`
   - 모듈 이해를 위한 `get_symbols_overview`
   - 주요 파일 찾기 위한 `find_file`

2. **흐름 분석용**:
   - 핵심 함수/클래스 이해를 위한 `find_symbol`
   - 의존성 매핑을 위한 `find_referencing_symbols`
   - 데이터 흐름 추적을 위한 `search_for_pattern`

3. **빠른 시작용**:
   - package.json, requirements.txt 등을 위한 `read_file`
   - 설정 명령어 테스트를 위한 `execute_shell_command`
   - 기존 설정 문서를 위한 메모리 확인

## 사용 예시

**사용자 요청**: "프로젝트 리뷰"

**프로세스**:
1. 메모리 나열 및 프로젝트 컨텍스트 확인
2. `list_dir`로 디렉토리 구조 분석
3. `get_symbols_overview`로 진입점 식별
4. project_brief.md 생성 (간결한 개요)
5. project_structure.md 생성 (주석이 달린 트리)
6. project_flow.md 생성 (심볼 분석을 사용한 mermaid 다이어그램)
7. quick_start.md 테스트 및 문서화 (모든 명령어 확인)
8. 모든 파일을 `/review` 디렉토리에 배치

**출력**: 즉시 사용할 수 있는 간결하고 사전 테스트된 4개의 마크다운 파일.
